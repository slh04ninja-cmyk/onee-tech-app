1. Vue d'ensemble

Cet Expert Advisor lit des fichiers CSV contenant des signaux de trading gold (exportés par Gold Signal Extractor) et les exécute dans le Strategy Tester de MT5.

CSV Files (dossier MQL5/Files/) ↓ GoldSignalBacktester.mq5 ↓ Lecture des signaux → Tri par date ↓ Pour chaque signal : ├─ Vérifier filtres (horaire, spread, quotidien) ├─ Déterminer le scénario (PU-S1, PU-S2, CAS1, CAS2-a, CAS2-b, QA) ├─ Exécuter (MARKET ou LIMIT) └─ Enregistrer le trade ↓ Gestion continue (chaque tick) : ├─ Break Even (BE) ├─ Gain Fixe (TP_FIXED) ├─ TP_TRIGGER (annulation pending) ├─ Expiration ordres └─ Limite quotidienne ↓ Rapport final par channel

Objectif

Comparer les channels Telegram pour trouver les meilleurs à utiliser dans le bot de copy trading (GZL TradingBot V9.0.0).

2. Installation

  1. Copier GoldSignalBacktester.mq5 dans MQL5/Experts/
  2. Copier les fichiers CSV dans MQL5/Files/
  3. Compiler dans MetaEditor
  4. Attacher au chart XAUUSD (ou XAUUSDm sur Exness)
  5. Ouvrir le Strategy Tester → sélectionner l'EA
Important : Le symbole du chart doit correspondre aux prix du CSV. Si les CSV contiennent des prix spot (XAUUSD), utilisez le chart XAUUSD. Si Exness, utilisez XAUUSDm.

4. Scénarios d'exécution

4.1 Prix Unique — S1 (MARKET)

Condition BUY : SL < prix < entry Condition SELL : entry < prix < SL Action : MARKET @ prix actuel Lot : LotUnique

4.2 Prix Unique — S2 (LIMIT)

Condition BUY : entry < prix < TP1 Condition SELL : TP1 < prix < entry Action : LIMIT @ entry Lot : LotUnique

4.3 CAS 1 — Prix dans la zone

Condition : zone_low ≤ prix ≤ zone_high Action : 50% lot → MARKET @ prix actuel 50% lot → LIMIT @ (zone + SL) / 2 Lot : LotTotal / 2

4.4 CAS 2-a — Prix entre TP1 et zone

Condition BUY : zone_high < prix < TP1 Condition SELL : TP1 < prix < zone_low Action : 50% lot → MARKET @ prix actuel 50% lot → LIMIT @ l'autre bout de la zone Lot : LotTotal / 2

4.5 CAS 2-b — Prix entre TP1 et TP2

Condition BUY : TP1 < prix < TP2 Condition SELL : TP2 < prix < TP1 Action : 50% lot → LIMIT @ zone_edge 50% lot → LIMIT @ zone_opposite Lot : LotTotal / 2

4.6 Quick Alert

Condition : Signal sans TP ni SL (ex: BUY 4512) Action : MARKET ou LIMIT SL : provisoire (SL_QuickAlert pts) TP : auto (RR_Ratio) Lot : LotUnique

5. Break Even (BE)

Quand le PnL de la pire position atteint PNL_Trigger points :

PNL min(positions) ≥ PNL_Trigger ? ├─ OUI → BE activé │ ├─ 1 position : SL @ entry_price │ ├─ 2 positions : SL @ (entry1 + entry2) / 2 │ ├─ Annuler ordres pending │ └─ target_gain = TP_FIXED_GAIN × N positions └─ NON → continuer
Whitelist BE : Seuls les rôles principaux déclenchent le BE (market_single, limit_single, market_cas1, market_cas2, limit_1, quick_market, quick_limit_filled, merge_limit). Les ordres pending (limit_2, limit_cas1, limit_cas2) ne déclenchent jamais le BE.

6. Gain Fixe (TP_FIXED)

Après le BE, le bot surveille le PnL total. Quand il atteint TP_FIXED_GAIN_USD × N positions, toutes les positions sont fermées manuellement.

BE activé avec 2 positions target_gain = 15.0 × 2 = 30.0$ PnL total ≥ 30.0$ → FERMETURE MANUELLE
Différence avec le broker TP : Le TP_FIXED ferme les positions dès que le gain est atteint, même si le prix n'a pas touché le TP du broker. Cela permet de sécuriser les gains plus tôt.

7. TP_TRIGGER

Quand le prix atteint le 3ème TP (ou le 2ème, ou le 1er si moins de 3 TPs) et qu'aucune position n'est ouverte, les ordres pending sont annulés.

Prix atteint TP[TPTrigger_Index] ? ├─ OUI → Annuler TOUS les ordres pending └─ NON → continuer
Cas typique : CAS 2-b avec 2 LIMIT. Le prix passe directement au TP3 sans remplir aucun ordre. Les LIMIT sont annulées car le mouvement est terminé.

8. Stop Loss configurable

Le SL peut être personnalisé avec des booléens d'activation :

ParamètreBooléenEffet
SL_CustomONSL calculé selon les distances max
SL_CustomOFFSL pris directement du signal CSV

Calcul du SL

SL_Custom = true : Prix Unique → SL = entry ± SL_PrixUnique pts Zone → SL = entry ± SL_PlusProche pts Quick Alert → SL = entry ± SL_QuickAlert pts SL_Custom = false : SL = valeur du CSV (telle quelle) Spread_Cost > 0 : SL ajusté + Spread_Cost pts (pour couvrir le coût du spread)

9. Spread & Slippage

9.1 Spread Filter

Quand Spread_Filter = true, les signaux dont le spread dépasse MaxSpreadPoints sont ignorés.

9.2 Slippage

Le slippage est appliqué via trade.SetDeviationInPoints(Slippage). MT5 simule le slippage dans le Strategy Tester.

9.3 Spread Cost

Quand Spread_Cost > 0, le SL est ajusté pour couvrir le coût du spread :

BUY : SL = SL - Spread_Cost (plus bas) SELL : SL = SL + Spread_Cost (plus haut)
Recommandation : Pour un backtesting réaliste, activez Spread_Filter avec MaxSpreadPoints=50 et Spread_Cost=5 pour couvrir le coût réel du spread sur Exness.
